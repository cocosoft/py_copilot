# 贡献指南

感谢您对 Py Copilot 项目感兴趣！我们欢迎各种形式的贡献，包括但不限于代码改进、错误修复、文档更新和功能建议。

## 行为准则

参与本项目的所有贡献者都应遵守基本的尊重和友善原则，共同维护一个积极健康的社区环境。

## 开始贡献

### 提交问题（Issues）

如果您发现了 bug 或有新功能的想法，请在 GitHub 上提交问题。提交问题时请包含：

- 清晰的标题
- 详细的描述
- 重现步骤（对于 bug）
- 期望行为与实际行为的对比
- 相关的环境信息（操作系统、浏览器版本等）
- 如有可能，添加截图或错误信息

### 提交代码

1. **Fork 仓库**
   在 GitHub 上 fork 项目到您自己的账号下。

2. **克隆仓库**
   ```bash
   git clone https://github.com/您的用户名/py-copilot.git
   cd py-copilot
   ```

3. **创建分支**
   为您的贡献创建一个新分支：
   ```bash
   git checkout -b feature/您的功能名称
   ```
   或
   ```bash
   git checkout -b fix/您要修复的问题
   ```

4. **安装依赖**
   按照 README.md 中的说明安装前后端依赖。

5. **编写代码**
   - 遵循项目现有的代码风格
   - 为新功能添加适当的测试
   - 确保现有测试通过

6. **提交更改**
   使用清晰、描述性的提交信息：
   ```bash
   git commit -m "简明扼要的描述：更详细的解释（如有必要）"
   ```

7. **推送到 GitHub**
   ```bash
   git push origin 您的分支名称
   ```

8. **创建 Pull Request**
   在 GitHub 上从您的分支创建一个 Pull Request 到主仓库的主分支。

## 代码规范

### 后端（Python）
- 遵循 PEP 8 编码规范
- 使用类型注解
- 添加适当的文档字符串
- 保持函数简洁，每个函数专注于单一职责

### 前端（JavaScript/React）
- 遵循 ESLint 规则
- 组件命名使用 PascalCase
- 函数和变量使用 camelCase
- 使用函数式组件和 Hooks
- 添加适当的注释

## 测试

提交代码前，请确保所有测试都能通过：

```bash
# 后端测试
cd backend
python -m pytest

# 前端测试（如果有）
cd frontend
npm test
```

## 文档

- 为新功能或更改添加文档
- 更新 README.md（如果适用）
- 保持文档清晰、简洁

## 审核流程

1. 维护者会审核您的 Pull Request
2. 可能会要求进行一些修改
3. 一旦通过审核，您的代码将被合并

## 获取帮助

如果您在贡献过程中需要帮助，可以通过以下方式获取支持：
- 在 GitHub Issues 中提问
- 通过项目的联系方式（如果提供）联系维护者

再次感谢您的贡献！
