# 大文件实体识别改造方案

## 1. 问题分析

### 1.1 当前问题
- 《三体》等大文件（1-3全集约90万字）导致实体识别失败
- 片段级视图显示实体数量为0，但文本内容正常显示
- 现有实体识别流程可能没有对大文件进行分段处理

### 1.2 根本原因
1. **内存限制**：一次性加载大文件进行实体识别，超出内存限制
2. **超时问题**：大文件处理时间过长，导致请求超时
3. **缺乏分片策略**：没有将大文件拆分为可管理的小片段进行分批处理
4. **错误处理不完善**：实体识别失败后没有提供重试机制
5. **前端显示问题**：片段级视图未正确调用片段实体API，导致显示为0

### 1.3 现有代码分析

#### 1.3.1 实体识别服务架构
```
KnowledgeGraphService (知识图谱服务)
├── extract_and_store_entities() - 全文档实体提取（问题所在）
├── get_document_entities() - 获取文档实体
└── get_document_relationships() - 获取文档关系

ChunkEntityService (片段级实体服务) ✅ 已存在
├── extract_chunk_entities() - 单片段实体识别 ✅
├── extract_document_entities_parallel() - 并行处理所有片段 ✅
└── get_chunk_entities() - 获取片段实体 ✅

DocumentProcessingService (文档处理服务)
├── process_document_entities() - 调用ChunkEntityService ✅
└── aggregate_document_entities() - 聚合文档实体 ✅
```

**关键发现：**
- `ChunkEntityService` 已经实现了片段级实体识别功能
- `extract_document_entities_parallel()` 支持并行处理多个片段
- 但前端没有正确调用这些API，而是调用了错误的API

#### 1.3.2 前端问题分析
```jsx
// FragmentLevelView.jsx 中的错误调用
// 错误：使用了 getChunkEntities（获取片段列表）
const response = await getChunkEntities(knowledgeBaseId, selectedChunkId, ...)

// 正确：应该使用 getChunkEntitiesDetail（获取片段实体）
const response = await getChunkEntitiesDetail(knowledgeBaseId, selectedChunkId, ...)
```

**问题总结：**
1. 后端已经有完善的片段级实体识别功能
2. 前端API调用错误，导致无法显示片段实体
3. 缺少"重新识别"功能，无法处理识别失败的情况
4. 缺少文档级聚合展示功能

## 2. 改造目标

1. **支持片段级实体识别**：在片段级视图中添加"重新识别"功能
2. **文档级聚合展示**：在文档级视图中聚合所有片段的实体
3. **大文件分片处理**：改造实体识别流程，支持大文件分片处理
4. **进度可视化**：显示实体识别进度和状态

### 3.3 实体识别流程优化

#### 3.3.1 文件大小判断

```python
class EntityExtractionConfig:
    """实体识别配置"""
    
    # 文件大小阈值（字节）
    SMALL_FILE_THRESHOLD = 100 * 1024      # 100KB
    MEDIUM_FILE_THRESHOLD = 1 * 1024 * 1024  # 1MB
    LARGE_FILE_THRESHOLD = 10 * 1024 * 1024  # 10MB
    
    # 分片大小配置
    CHUNK_SIZE_SMALL = 500   # 小文件分片大小
    CHUNK_SIZE_MEDIUM = 1000  # 中等文件分片大小
    CHUNK_SIZE_LARGE = 2000   # 大文件分片大小

class FileSizeAwareExtractor:
    """文件大小感知的实体提取器"""
    
    def get_extraction_strategy(self, file_size: int) -> ExtractionStrategy:
        """
        根据文件大小选择提取策略：
        - 小文件(<100KB): 直接处理，不分片
        - 中等文件(100KB-1MB): 固定长度分片
        - 大文件(1MB-10MB): 段落优先分片
        - 超大文件(>10MB): 混合分片策略 + 异步处理
        """
        if file_size < EntityExtractionConfig.SMALL_FILE_THRESHOLD:
            return DirectExtractionStrategy()
        elif file_size < EntityExtractionConfig.MEDIUM_FILE_THRESHOLD:
            return FixedChunkStrategy(chunk_size=1000)
        elif file_size < EntityExtractionConfig.LARGE_FILE_THRESHOLD:
            return ParagraphChunkStrategy(max_chunk_size=2000)
        else:
            return AsyncBatchStrategy(
                chunk_size=2000,
                batch_size=10,
                use_celery=True  # 使用Celery异步处理
            )
```

#### 3.3.2 异步处理大文件

```python
# 使用Celery进行异步处理
from celery import Celery

app = Celery('entity_extraction')

@app.task(bind=True, max_retries=3)
def extract_entities_async(self, document_id: int, chunk_ids: List[int]):
    """
    异步提取实体任务
    
    特性：
    1. 支持进度报告
    2. 自动重试（最多3次）
    3. 失败片段记录
    """
    try:
        extractor = EntityExtractor()
        results = []
        
        for i, chunk_id in enumerate(chunk_ids):
            # 报告进度
            self.update_state(
                state='PROGRESS',
                meta={
                    'current': i + 1,
                    'total': len(chunk_ids),
                    'percent': int((i + 1) / len(chunk_ids) * 100)
                }
            )
            
            # 处理单个片段
            result = extractor.extract_from_chunk(chunk_id)
            results.append(result)
        
        # 聚合结果
        aggregate_results(document_id, results)
        
        return {
            'status': 'completed',
            'total_chunks': len(chunk_ids),
            'successful': len([r for r in results if r.success]),
            'failed': len([r for r in results if not r.success])
        }
    
    except Exception as exc:
        # 重试逻辑
        if self.request.retries < self.max_retries:
            raise self.retry(exc=exc, countdown=60)
        raise
```
