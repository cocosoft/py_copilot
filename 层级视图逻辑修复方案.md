# 层级视图逻辑修复方案

## 一、问题分析

当前实现存在以下核心问题：

### 1.1 数据层级混乱
- 四个层级都显示相同的实体数量（87个）
- 片段级没有片段选择功能
- 文档级没有文档选择功能
- 统计数据显示不一致

### 1.2 缺少层级特有的数据筛选
- **片段级**：应该只显示选中片段的实体
- **文档级**：应该只显示选中文档的实体
- **知识库级**：应该显示当前知识库的聚合统计
- **全局级**：应该显示跨知识库的数据

### 1.3 API 接口不完善
- 缺少获取文档列表的接口
- 缺少获取片段列表的接口
- 缺少按层级筛选实体的接口

## 二、层级关系梳理

### 2.1 数据层级结构

```
全局级 (Global)
    ├── 知识库1 (Knowledge Base) ← 最基础的容器
    │       ├── 文档1 (Document)
    │       │       ├── 片段1 (Chunk)
    │       │       ├── 片段2 (Chunk)
    │       │       └── 片段3 (Chunk)
    │       ├── 文档2 (Document)
    │       │       ├── 片段1 (Chunk)
    │       │       └── 片段2 (Chunk)
    │       └── 文档3 (Document)
    │               ├── 片段1 (Chunk)
    │               └── 片段2 (Chunk)
    ├── 知识库2 (Knowledge Base)
    │       └── ...
    └── 知识库3 (Knowledge Base)
            └── ...
```

**关键原则：**
- **知识库是最基础的容器**，没有知识库就没有文档
- 文档必须属于某个知识库
- 片段必须属于某个文档
- 全局级可以跨知识库查看

### 2.2 各层级数据范围

| 层级 | 数据范围 | 选择逻辑 | 前提条件 | 数据粒度 |
|------|----------|----------|----------|----------|
| **片段级** | 单个片段的实体 | 选知识库 → 选文档 → 选片段 → 显示该片段实体 | 必须先选知识库，再选文档，再选片段 | 最细粒度 |
| **文档级** | 单个文档的所有实体（包含该文档所有片段） | 选知识库 → 选文档 → 显示该文档所有片段的实体聚合 | 必须先选知识库，再选文档 | 中等粒度 |
| **知识库级** | 整个知识库的实体（包含所有文档） | 选知识库 → 显示该知识库所有文档的实体聚合 | 必须先选知识库 | 粗粒度 |
| **全局级** | 跨知识库的实体（包含所有知识库） | 直接进入 → 显示所有知识库的实体聚合 | 无前提条件 | 最粗粒度 |

### 2.3 数据流向与聚合关系

```
【片段级 - 最细粒度】
选择知识库 → 选择文档A → 加载文档A的片段列表 → 选择片段A-1 → 加载片段A-1的实体
显示：片段A-1的实体（如：5个）

【文档级 - 中等粒度】
选择知识库 → 选择文档A → 加载文档A的所有实体（包含片段A-1、A-2、A-3的所有实体）
显示：文档A的实体（如：25个 = 片段A-1的5个 + 片段A-2的10个 + 片段A-3的10个）

【知识库级 - 粗粒度】
选择知识库 → 加载该知识库所有文档的实体
显示：知识库的实体（如：1,200个 = 文档A的25个 + 文档B的30个 + ...）

【全局级 - 最粗粒度】
直接进入 → 加载所有知识库的实体
显示：全局实体（如：3,000个 = 知识库1的1,200个 + 知识库2的800个 + ...）
```

**重要说明：**
- 知识库是最基础的容器，所有操作都必须基于某个知识库（全局级除外）
- 当前已在某个知识库页面时，知识库选择可以省略，默认使用当前知识库
- 各层级的数据是逐层聚合的关系：片段 → 文档 → 知识库 → 全局

## 三、修复方案

### 3.1 后端 API 完善

#### 新增 API 接口

##### 1. 获取知识库的文档列表
```
GET /api/v1/knowledge/knowledge-bases/{knowledge_base_id}/documents

Response:
{
  "data": [
    {
      "id": 1,
      "title": "文档标题",
      "entityCount": 25,
      "relationCount": 18,
      "chunkCount": 10,
      "createdAt": "2026-01-01"
    }
  ],
  "total": 50
}
```

##### 2. 获取文档的片段列表
```
GET /api/v1/knowledge/knowledge-bases/{knowledge_base_id}/documents/{document_id}/chunks

Response:
{
  "data": [
    {
      "id": 1,
      "text": "片段文本内容...",
      "startPosition": 0,
      "endPosition": 500,
      "entityCount": 5,
      "index": 1
    }
  ],
  "total": 10
}
```

##### 3. 获取指定片段的实体
```
GET /api/v1/knowledge/knowledge-bases/{knowledge_base_id}/chunks/{chunk_id}/entities

Response:
{
  "data": [实体列表],
  "total": 5
}
```

##### 4. 获取指定文档的实体
```
GET /api/v1/knowledge/knowledge-bases/{knowledge_base_id}/documents/{document_id}/entities

Response:
{
  "data": [实体列表],
  "total": 25
}
```

### 2.3 数据流设计

```
用户操作 → 状态管理 → API调用 → 数据筛选 → 视图更新

【片段级数据流】
1. 用户选择文档
   ↓
2. 更新 selectedDocumentId 状态
   ↓
3. 调用 getDocumentChunks API 获取片段列表
   ↓
4. 清空 selectedChunkId，显示片段选择器
   ↓
5. 用户选择片段
   ↓
6. 更新 selectedChunkId 状态
   ↓
7. 调用 getChunkEntities API 获取片段实体
   ↓
8. 更新视图显示

【文档级数据流】
1. 用户选择文档
   ↓
2. 更新 selectedDocumentId 状态
   ↓
3. 调用 getDocumentEntities API 获取文档实体
   ↓
4. 更新视图显示

【知识库级数据流】
1. 进入知识库级视图
   ↓
2. 调用 getKnowledgeBaseStats API 获取知识库统计
   ↓
3. 更新视图显示

【全局级数据流】
1. 进入全局级视图
   ↓
2. 调用 getGlobalStats API 获取全局统计
   ↓
3. 更新视图显示
```

## 四、关键设计决策

### 4.1 数据加载策略
- **懒加载**：选择文档后才加载片段列表
- **缓存机制**：已加载的文档列表和片段列表进行缓存

### 4.2 状态管理
- 使用 Zustand 管理层级状态
- 选择状态保存在 URL 参数中，支持刷新后恢复

### 4.3 错误处理
- 每个层级独立处理错误
- 提供友好的错误提示和重试机制

### 4.4 性能优化
- **虚拟列表**：大量实体时使用虚拟滚动
- **分页加载**：实体列表分页显示
- **防抖处理**：搜索和筛选时防抖

## 五、预期效果

### 5.1 片段级（最细粒度）
- **选择流程**：用户先选择文档 → 再选择片段 → 显示该片段的实体
- **数据范围**：只显示选中片段的实体
- **统计显示**：该片段的实体数量（如：5个实体）
- **关键约束**：必须先选择文档，才能选择该文档下的片段

### 5.2 文档级（中等粒度）
- **选择流程**：用户选择文档 → 显示该文档的实体和关系
- **数据范围**：显示该文档**所有片段**的实体聚合
- **统计显示**：该文档的总实体数量（如：25个实体 = 片段1的5个 + 片段2的10个 + 片段3的10个）
- **关键约束**：不同文档显示不同的实体数据

### 5.3 知识库级（粗粒度）
- **选择流程**：进入知识库级 → 显示当前知识库的整体统计
- **数据范围**：显示该知识库**所有文档**的实体聚合
- **统计显示**：
  - 文档总数：50
  - 实体总数：1,200（未去重）/ 850（去重后）
  - 关系总数：800
  - 各文档的实体分布统计

### 5.4 全局级（最粗粒度）
- **选择流程**：进入全局级 → 显示跨知识库的全局统计
- **数据范围**：显示**所有知识库**的实体聚合
- **统计显示**：
  - 知识库总数：3
  - 文档总数：50
  - 实体总数：3,000（跨库去重后）
  - 各知识库的对比统计
