# 层级视图逻辑修复方案实施计划

## 一、项目概述

### 1.1 项目背景
当前层级视图存在数据层级混乱问题：四个层级都显示相同的实体数量（87个），片段级和文档级缺少相应的选择功能，统计数据显示不一致。

### 1.2 项目目标
- 修复四个层级（片段级、文档级、知识库级、全局级）的数据筛选逻辑
- 实现各层级特有的数据展示和统计功能
- 完善后端API接口，支持按层级筛选实体
- 优化前端交互体验，添加必要的文档和片段选择器

### 1.3 项目范围
- **后端**：新增4个API接口，修复统计逻辑
- **前端**：重构4个层级视图组件，新增2个选择器组件，扩展状态管理

### 5.3 应急预案

**场景1：API性能不达标**
- 触发条件：实体列表加载时间>2秒
- 应对措施：
  1. 增加数据库索引
  2. 使用Redis缓存热点数据
  3. 优化查询SQL
  4. 如仍不达标，考虑降级方案（限制单次查询数量）

**场景2：上线后出现严重缺陷**
- 触发条件：生产环境出现P0级别缺陷
- 应对措施：
