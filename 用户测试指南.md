## 功能测试步骤

### 测试1：创建知识库

**API端点**: `POST /api/v1/knowledge/bases`

**请求体**:
```json
{
  "name": "测试知识库",
  "description": "用于测试的知识库"
}
```

**预期结果**:
- 状态码: 200 或 201
- 返回包含知识库ID的JSON

---

### 测试2：上传文档

**API端点**: `POST /api/v1/knowledge/bases/{knowledge_base_id}/documents`

**请求方式**: FormData
- 字段名: `file`
- 文件类型: .txt, .pdf, .docx 等

**测试步骤**:
1. 准备一个测试文档（如test.txt），内容如下：
```
人工智能（AI）是计算机科学的一个分支。
机器学习是AI的重要子领域。
深度学习使用神经网络处理复杂数据。
自然语言处理让计算机理解人类语言。
```

2. 使用curl上传：
```bash
curl -X POST "http://localhost:8007/api/v1/knowledge/bases/{knowledge_base_id}/documents" \
  -F "file=@test.txt"
```

**预期结果**:
- 状态码: 200 或 201
- 返回包含文档ID的JSON
- 文档状态为"processing"

---

### 测试3：检查文档处理状态

**API端点**: `GET /api/v1/knowledge/bases/{knowledge_base_id}/documents/{document_id}`

**预期结果**:
- 状态码: 200
- `processing_status` 字段最终变为 "completed"
- `vectorization_rate` 显示向量化成功率（应接近100%）
- `chunks_count` 显示分块数量

---

### 测试4：搜索功能

**API端点**: `POST /api/v1/knowledge/bases/{knowledge_base_id}/search`

**请求体**:
```json
{
  "query": "什么是人工智能",
  "limit": 5
}
```

**预期结果**:
- 状态码: 200
- 返回相关文档列表
- 结果按相关性排序（已启用重排序）

---

### 测试5：知识图谱

**API端点**: `GET /api/v1/knowledge/bases/{knowledge_base_id}/graph`

**预期结果**:
- 状态码: 200
- 返回包含nodes（实体节点）和edges（关系边）的JSON
- 实体识别使用spaCy NER

## 常见问题

### Q1: 文档上传后一直显示"processing"
**解决方案**:
- 检查后端日志是否有错误
- 等待1-2分钟（首次处理需要时间）
- 刷新页面查看状态更新

### Q2: 搜索结果不准确
**解决方案**:
- 确保文档已完成向量化
- 检查查询词是否与文档内容相关
- 重排序模型需要加载后才能生效

### Q3: 知识图谱为空
**解决方案**:
- 确保文档包含可识别的实体
- 检查spaCy模型是否正确加载
- 查看后端日志确认实体识别是否执行

### Q4: 服务启动慢
**原因**:
- 首次启动需要加载模型
- 模型加载时间约10-30秒
- 后续启动会更快
