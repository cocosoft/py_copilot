# 知识图谱前端优化方案

## 一、现状分析

### 1.1 当前功能结构

通过代码分析，发现当前知识图谱相关功能分布在以下几个层级：

#### 全局级别（Global）
- **实体配置管理** (`EntityConfigManagement.jsx`)
  - 实体类型管理（添加、编辑、启用/禁用）
  - 提取规则管理（正则规则配置）
  - 词典管理（术语词典维护）
  - 实体提取测试

- **实体维护** (`EntityMaintenanceSimple.jsx`)
  - 全局实体列表查看
  - 实体编辑、删除
  - 批量操作

#### 知识库级别（Knowledge Base）
- **知识库知识图谱** (在 `Knowledge.jsx` 中)
  - 构建整个知识库的知识图谱
  - 查看知识库级别的图谱可视化
  - 知识库级别的图谱统计

- **知识库实体维护** (在 `EntityMaintenanceSimple.jsx` 中)
  - 查看知识库内的实体列表
  - 知识库级别的实体统计

#### 文档级别（Document）
- **文档知识图谱** (在 `Knowledge.jsx` 中)
  - 构建单个文档的知识图谱
  - 查看文档级别的图谱可视化
  - 文档级别的图谱统计和分析

### 1.2 存在的问题

#### 1.2.1 层级关系混乱
```
问题表现：
- 用户不清楚"全局实体"、"知识库实体"、"文档实体"之间的关系
- 实体配置是全局的，但实体维护却分成了全局和知识库两个视图
- 知识图谱构建在文档详情页和知识库页面都有入口，但功能不一致
```

#### 1.2.2 功能入口分散
```
问题表现：
- 实体配置在独立的管理页面
- 实体维护在另一个独立页面
- 知识图谱可视化在知识库页面和文档详情页
- 缺少统一的知识图谱管理中心
```

#### 1.2.3 实体配置功能薄弱
```
问题表现：
- 实体类型配置只支持简单的名称、描述、颜色
- 缺少实体属性配置（如：人物的职位、公司的行业等）
- 提取规则只支持正则，不支持LLM提示词配置
- 缺少实体关系类型配置
- 没有实体质量评分机制
```

#### 1.2.4 实体维护功能不足
```
问题表现：
- 只能查看和删除实体，缺少合并、拆分功能
- 没有实体消歧功能（同一实体的不同表述）
- 缺少实体链接功能（链接到外部知识库如维基百科）
- 没有实体审核工作流
- 缺少实体变更历史记录
```

#### 1.2.5 知识图谱可视化单一
```
问题表现：
- 只有基本的力导向图展示
- 缺少多种视图模式（如：层次图、时间线图、地理图）
- 没有图谱探索功能（路径发现、社区发现）
- 缺少实体详情侧边栏
- 不支持图谱编辑（添加/删除节点和边）
```

#### 1.2.6 缺少批量操作
```
问题表现：
- 只能逐个构建文档的知识图谱
- 缺少批量构建功能
- 没有批量实体操作（批量合并、批量分类）
```

## 二、优化目标

### 2.1 核心目标
1. **统一管理层级**：明确区分全局、知识库、文档三个层级的管理范围
2. **集中管理入口**：建立统一的知识图谱管理中心
3. **增强配置能力**：支持更丰富的实体类型和关系类型配置
4. **完善维护功能**：提供实体全生命周期管理
5. **提升可视化**：提供多种视图和交互方式
6. **支持批量操作**：提高大规模数据管理效率

### 2.2 用户体验目标
- 降低知识图谱管理的学习成本
- 提高实体维护的效率
- 提供直观的图谱探索体验
- 支持灵活的自定义配置

#### 3.1.2 页面结构重组

建议将知识图谱相关功能整合到一个独立的"知识图谱"模块：

```
侧边栏导航
├── 知识库（现有）
│   └── 文档管理、搜索等功能
├── 知识图谱（新增/重构）
│   ├── 全局配置
│   ├── 知识库图谱
│   └── 文档图谱
└── ...其他模块
```

### 3.2 功能增强

#### 3.2.1 实体类型配置增强

**当前功能**：
- 名称、描述、颜色

**增强功能**：
```typescript
interface EntityTypeConfig {
  // 基础信息
  id: string;
  name: string;
  description: string;
  color: string;
  icon: string;  // 新增：图标
  
  // 扩展属性定义
  properties: PropertyDef[];
  
  // 识别规则
  recognitionRules: {
    regex?: string[];           // 正则规则
    dictionary?: string[];      // 词典
    llmPrompt?: string;         // LLM提示词
    examples?: string[];        // 示例
  };
  
  // 消歧配置
  disambiguation: {
    enabled: boolean;
    contextWindow: number;      // 上下文窗口大小
    similarityThreshold: number; // 相似度阈值
  };
  
  // 质量规则
  qualityRules: {
    minConfidence: number;
    requiredProperties: string[];
    validationRules: ValidationRule[];
  };
}

interface PropertyDef {
  name: string;
  type: 'string' | 'number' | 'date' | 'enum' | 'url' | 'entity';
  required: boolean;
  description: string;
  enumValues?: string[];        // 当type为enum时
  linkedEntityType?: string;    // 当type为entity时
}
```

#### 3.2.2 关系类型配置

**新增功能**：
```typescript
interface RelationTypeConfig {
  id: string;
  name: string;
  description: string;
  
  // 方向性
  directional: boolean;
  
  // 允许的实体类型
  sourceEntityTypes: string[];
  targetEntityTypes: string[];
  
  // 关系属性
  properties: PropertyDef[];
  
  // 提取规则
  extractionRules: {
    patterns: string[];         // 关系模式（如："{PERSON}是{ORG}的CEO"）
    llmPrompt?: string;
    keywords: string[];         // 触发关键词
  };
  
  // 约束规则
  constraints: {
    unique: boolean;            // 是否唯一（如：一个人只有一个出生地）
    transitive: boolean;        // 是否可传递
    symmetric: boolean;         // 是否对称
  };
}
```

#### 3.2.3 实体维护功能增强

**实体列表增强**：
- 高级筛选（按类型、属性、来源、质量评分等）
- 批量操作（批量删除、批量合并、批量修改类型）
- 智能推荐（推荐合并的相似实体）
- 冲突标记（标记有冲突的实体）

**实体详情增强**：
```typescript
interface EntityDetail {
  // 基础信息
  id: string;
  name: string;
  type: string;
  confidence: number;
  
  // 属性
  properties: Record<string, any>;
  
  // 来源
  sources: {
    documentId: string;
    documentName: string;
    positions: Array<{start: number, end: number, text: string}>;
    confidence: number;
  }[];
  
  // 关系
  relations: {
    outgoing: Relation[];
    incoming: Relation[];
  };
  
  // 消歧信息
  disambiguation: {
    canonicalId: string;        // 标准实体ID
    aliases: string[];          // 别名
    similarEntities: SimilarEntity[];
  };
  
  // 质量信息
  quality: {
    score: number;
    issues: QualityIssue[];
    history: QualityHistory[];
  };
  
  // 外部链接
  externalLinks: {
    wikipedia?: string;
    wikidata?: string;
    custom: Array<{name: string, url: string}>;
  };
  
  // 变更历史
  history: ChangeHistory[];
}
```

**实体合并功能**：
- 相似实体检测（基于名称相似度、上下文相似度）
- 合并预览（展示合并前后的变化）
- 冲突解决（处理属性冲突）
- 批量合并（一次合并多个实体）

**实体消歧功能**：
- 同名实体列表
- 上下文对比
- 手动指定标准实体
- 自动消歧建议

#### 3.2.4 知识图谱可视化增强

**多视图模式**：
1. **力导向图**：展示整体结构，支持社区着色
2. **层次图**：按实体类型或社区分层展示
3. **时间线图**：展示带时间属性的实体演变
4. **地理图**：在地图上展示带位置属性的实体
5. **径向图**：以选中实体为中心展示关联

**交互增强**：
- 实体详情侧边栏（点击节点显示详情）
- 图谱编辑（添加/删除节点和边）
- 路径高亮（高亮两个实体间的路径）
- 社区高亮
- 缩放和平滑动画
- 搜索定位

**分析功能**：
- 社区发现（自动识别实体社区）
- 中心性分析（识别关键实体）
- 路径发现（查找两个实体间的最短路径）
- 异常检测（识别孤立的、矛盾的实体）

#### 3.2.5 批量操作功能

**批量构建**：
- 选择多个文档进行批量构建
- 进度显示和取消功能
- 错误处理和重试

**批量实体操作**：
- 批量修改实体类型
- 批量合并相似实体
- 批量删除低质量实体
- 批量导出/导入
