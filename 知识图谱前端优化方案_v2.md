## 一、现有架构分析

### 1.1 设置模块架构
```
Settings.jsx（设置主页面）
├── 左侧导航菜单（nav-item）
│   ├── general（通用设置）
│   ├── model（模型管理）
│   ├── knowledge（知识库）
│   ├── workflow（工作流）
│   └── globalMemory（全局记忆）
│
└── 右侧内容区域（renderContent）

    ├── IntegratedModelManagement（模型管理）
```

**复用点**：
- 侧边栏导航样式（`settings-sidebar`、`nav-item`）

## 二、优化后的架构设计

### 2.1 整体架构

采用"**设置集中配置 + 知识库分级管理**"的双层架构：

```
┌─────────────────────────────────────────────────────────────┐
│                     设置 - 知识图谱配置                       │
│  （全局实体类型、关系类型、提取策略、质量规则配置）              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     知识库 - 知识图谱管理                     │
│  ┌──────────────┬──────────────┬──────────────┐              │
│  │  知识库级别   │   文档级别    │   实体维护   │              │
│  │  (KB Level)  │ (Doc Level)  │(Maintenance) │              │
│  └──────────────┴──────────────┴──────────────┘              │
└─────────────────────────────────────────────────────────────┘
```

```jsx
// components/KnowledgeGraph/KnowledgeGraphConfig.jsx
import React, { useState } from 'react';
import EntityConfigManagement from '../EntityConfigManagement';
import RelationTypeConfig from './RelationTypeConfig';
import ExtractionStrategyConfig from './ExtractionStrategyConfig';
import QualityRuleConfig from './QualityRuleConfig';
import './KnowledgeGraphConfig.css';

const KnowledgeGraphConfig = () => {

    <div className="knowledge-graph-config">

      <div className="config-tabs">
        {tabs.map(tab => (
          <button
            key={tab.id}
            className={`tab-btn ${activeTab === tab.id ? 'active' : ''}`}
            onClick={() => setActiveTab(tab.id)}
          >

        {activeTab === 'relation-types' && <RelationTypeConfig />}
        {activeTab === 'extraction' && <ExtractionStrategyConfig />}
```

```javascript
// utils/api/knowledgeGraphApi.js

/**
 * 获取关系类型列表
 */
export const getRelationTypes = async () => {
  return request('/v1/knowledge-graph/relation-types', { method: 'GET' });
};

/**
 * 添加关系类型
 */
export const addRelationType = async (data) => {
  return request('/v1/knowledge-graph/relation-types', { 
    method: 'POST', 
    data 
  });
};

/**
 * 批量构建知识图谱
 */
export const batchBuildKnowledgeGraph = async (documentIds, options = {}) => {
  return request('/v1/knowledge-graph/batch/build-graphs', {
    method: 'POST',
    data: { document_ids: documentIds, ...options }
  });
};

/**
 * 合并实体
 */
export const mergeEntities = async (sourceIds, targetId) => {
  return request('/v1/knowledge-graph/entities/merge', {
    method: 'POST',
    data: { source_ids: sourceIds, target_id: targetId }
  });
};

/**
 * 获取相似实体
 */
export const getSimilarEntities = async (entityId, threshold = 0.8) => {
  return request(`/v1/knowledge-graph/entities/${entityId}/similar?threshold=${threshold}`);
};
```
