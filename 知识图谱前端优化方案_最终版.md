# 知识图谱前端优化方案（最终版）

## 一、现状问题总结

### 1.1 核心问题

通过代码分析和用户反馈，当前知识图谱功能存在以下核心问题：

| 问题类别 | 具体问题 | 影响 |
|---------|---------|------|
| **层级混乱** | 全局/知识库/文档三个层级的实体管理界限不清 | 用户困惑，操作错误 |
| **入口分散** | 实体配置、实体维护、知识图谱可视化分布在不同页面 | 操作繁琐，效率低下 |
| **配置薄弱** | 仅支持基础实体类型配置，缺少关系类型、LLM提示词、质量规则配置 | 无法满足复杂场景需求 |
| **维护不足** | 缺少实体合并、消歧、审核等功能 | 实体质量难以保证 |
| **可视化单一** | 仅有力导向图，缺少多视图和分析功能 | 探索和分析能力弱 |
| **批量缺失** | 只能逐个构建文档知识图谱 | 大规模处理效率低 |

### 1.2 用户场景痛点

```
场景1：配置实体类型
现状：需要在独立页面配置，配置后不知道效果如何
痛点：配置-测试-调整流程不闭环

场景2：管理知识库实体
现状：实体列表和图谱可视化分离
痛点：无法直观看到实体间关系，维护困难

场景3：处理文档实体
现状：只能查看，无法确认或修改
痛点：提取错误无法纠正，影响图谱质量

场景4：批量构建图谱
现状：只能逐个文档构建
痛点：知识库文档多时操作繁琐
```

## 二、优化目标

### 2.1 核心目标

1. **统一管理层级**：明确区分全局配置、知识库管理、文档处理三个层级
2. **集中管理入口**：建立统一的知识图谱管理中心
3. **增强配置能力**：支持实体类型、关系类型、提取策略、质量规则的全方位配置
4. **完善维护功能**：提供实体全生命周期管理（提取→确认→维护→优化）
5. **提升可视化**：提供多种视图和交互方式，支持图谱探索和分析
6. **支持批量操作**：实现高效的批量构建和批量维护

### 2.2 用户体验目标

- **降低学习成本**：清晰的层级划分，符合直觉的操作流程
- **提高操作效率**：批量操作、智能推荐、快捷编辑
- **增强可视化体验**：多维度展示，支持探索式分析
- **支持灵活配置**：满足不同业务场景的自定义需求

const KnowledgeGraphContext = createContext();

const initialState = {

  switch (action.type) {
    case 'SET_ENTITY_TYPES':
      return { ...state, entityTypes: action.payload };

      return { ...state, entities: action.payload };

  }

export const useKnowledgeGraphOperation = (operation) => {
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const execute = async (...args) => {
    setLoading(true);
    setError(null);
    
    try {
      const result = await operation(...args);
      return result;
    } catch (err) {
      setError(err.message);
      throw err;
    } finally {
      setLoading(false);
    }
  };

  return { execute, loading, error };
};
