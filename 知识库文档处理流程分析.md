### 3.3 文本清理与预处理

**核心类：** `AdvancedTextProcessor`

**处理步骤：**
1. **去除噪声字符** - 删除特殊符号、多余空白
2. **统一编码** - 确保UTF-8编码
3. **格式标准化** - 统一段落、列表格式
4. **语言检测** - 识别文本语言类型

```python
def clean_text(self, text: str) -> str:
    """
    清理和预处理文本
    
    处理内容：
    - 去除HTML标签
    - 规范化空白字符
    - 修复编码问题
    - 去除特殊控制字符
    """
    if not text:
        return ""
    
    # 去除HTML标签
    text = re.sub(r'<[^>]+>', '', text)
    
    # 规范化空白
    text = re.sub(r'\s+', ' ', text)
    
    # 去除控制字符
    text = re.sub(r'[\x00-\x08\x0b-\x0c\x0e-\x1f]', '', text)
    
    return text.strip()
```

### 3.4 智能分块

**核心类：** `DocumentProcessor._simple_chunking()`

**分块策略：**
- **基于规则的分块** - 使用段落和句子边界
- **语义分块** - 保持语义完整性
- **重叠处理** - 确保上下文连续性

**分块参数：**
```python
max_chunk_size: int = 1000    # 最大块大小（字符数）
min_chunk_size: int = 200     # 最小块大小（字符数）
overlap: int = 50             # 块之间的重叠大小
```

**分块算法：**
```python
def _simple_chunking(self, text: str, max_chunk_size: int = 1000,
                     min_chunk_size: int = 200, overlap: int = 50) -> List[str]:
    """
    基于规则的简单分块方法
    
    策略：
    1. 优先在段落边界分割
    2. 长段落按句子分割
    3. 保持块间重叠确保连续性
    """
    chunks = []
    
    # 首先按段落分割
    paragraphs = re.split(r'\n\s*\n', text)
    
    current_chunk = ""
    for paragraph in paragraphs:
        if len(current_chunk) + len(paragraph) <= max_chunk_size:
            current_chunk += "\n\n" + paragraph if current_chunk else paragraph
        else:
            # 保存当前块
            if len(current_chunk) >= min_chunk_size:
                chunks.append(current_chunk.strip())
                # 保留重叠部分
                current_chunk = current_chunk[-overlap:] if overlap > 0 else ""
            
            # 处理长段落
            if len(paragraph) > max_chunk_size:
                sentences = re.split(r'(?<=[.!?。！？])\s+', paragraph)
                for sentence in sentences:
                    if len(current_chunk) + len(sentence) <= max_chunk_size:
                        current_chunk += " " + sentence if current_chunk else sentence
                    else:
                        if current_chunk:
                            chunks.append(current_chunk.strip())
                        current_chunk = sentence
            else:
                current_chunk = paragraph
    
    # 保存最后一个块
    if current_chunk:
        chunks.append(current_chunk.strip())
    
    return chunks
```

### 3.5 向量化处理

**核心类：** `ChromaService`

**架构设计：**
- 独立ChromaDB服务（端口8008）
- HTTP API通信
- 批量处理优化

**向量化流程：**
```python
def process_document(self, file_path: str, file_type: str, 
                     document_id: int, knowledge_base_id: int) -> Dict[str, Any]:
    """
    文档向量化处理流程
    
    步骤：
    1. 文档解析
    2. 文本清理
    3. 智能分块
    4. 向量化处理
    """
    # 1. 解析文档
    raw_text = self.parser.parse_document(file_path)
    
    # 2. 文本预处理
    cleaned_text = self.text_processor.clean_text(raw_text)
    
    # 3. 智能分块
    chunks = self._simple_chunking(cleaned_text)
    
    # 4. 批量向量化
    batch_size = 100 if len(chunks) > 100 else len(chunks)
    for batch_idx in range(0, len(chunks), batch_size):
        batch_chunks = chunks[batch_idx:batch_idx + batch_size]
        
        # 准备批量数据
        batch_documents = []
        for i, chunk in enumerate(batch_chunks):
            chunk_id = f"{document_id}_chunk_{batch_idx + i}"
            metadata = {
                "document_id": document_id,
                "knowledge_base_id": knowledge_base_id,
                "chunk_index": batch_idx + i,
                "total_chunks": len(chunks)
            }
            batch_documents.append({
                "document_id": chunk_id,
                "text": chunk,
                "metadata": metadata
            })
        
        # 批量添加到向量数据库
        self.chroma_service.add_documents_batch(batch_documents)
```

**ChromaService API：**
```python
class ChromaService:
    def __init__(self, server_url: str = "http://localhost:8008"):
        self.server_url = server_url
        self.session = create_retry_session(retries=3, timeout=60)
    
    def add_documents_batch(self, documents: List[Dict]) -> Dict[str, Any]:
        """批量添加文档到向量数据库"""
        response = self.session.post(
            f"{self.server_url}/collections/documents/batch",
            json={"documents": documents},
            timeout=180
        )
        return response.json()
    
    def search_similar(self, query: str, top_k: int = 5) -> List[Dict]:
        """相似度搜索"""
        response = self.session.post(
            f"{self.server_url}/collections/documents/search",
            json={"query": query, "top_k": top_k}
        )
        return response.json().get("results", [])
```

### 4.3 实体识别流程

#### 4.3.1 片段级实体识别

**核心类：** `ChunkEntityService`

**处理流程：**
```python
def extract_chunk_entities(self, chunk_id: int,
                           knowledge_base_id: int = None) -> Dict[str, Any]:
    """
    对单个片段进行实体识别

    复用：LLMEntityExtractor.extract_entities()

    Args:
        chunk_id: 片段ID
        knowledge_base_id: 知识库ID（用于获取配置）

    Returns:
        {"chunk_id": int, "entity_count": int, "entities": List}
    """
    try:
        # 获取片段
        chunk = self.db.query(DocumentChunk).filter(
            DocumentChunk.id == chunk_id
        ).first()

        if not chunk:
            return {"chunk_id": chunk_id, "entity_count": 0, "error": "片段不存在"}

        # 复用现有LLM实体提取器
        extractor = LLMEntityExtractor(
            self.db,
            knowledge_base_id=knowledge_base_id or chunk.document.knowledge_base_id
        )

        # 提取实体（仅处理片段文本）
        try:
            entities = asyncio.run(extractor.extract_entities(chunk.chunk_text))
        except RuntimeError as e:
            # 处理事件循环冲突
            if "cannot be called from a running event loop" in str(e):
                # 使用 nest_asyncio 或线程池处理
                import nest_asyncio
                nest_asyncio.apply()
                entities = asyncio.run(extractor.extract_entities(chunk.chunk_text))

        # 保存为ChunkEntity
        chunk_entities = []
        for entity in entities:
            chunk_entity = ChunkEntity(
                chunk_id=chunk_id,
                document_id=chunk.document_id,
                entity_text=entity.get("text", ""),
                entity_type=entity.get("type", "UNKNOWN"),
                start_pos=entity.get("start_pos", 0),
                end_pos=entity.get("end_pos", 0),
                confidence=entity.get("confidence", 0.8),
                context=entity.get("context", "")
            )
            self.db.add(chunk_entity)
            chunk_entities.append(chunk_entity)

        self.db.commit()

        return {
            "chunk_id": chunk_id,
            "entity_count": len(chunk_entities),
            "entities": [{"id": e.id, "text": e.entity_text, "type": e.entity_type}
                        for e in chunk_entities]
        }
```

```python
def _build_entity_extraction_prompt(self, text: str) -> str:
    """
    构建实体提取的Prompt

    根据配置动态构建Prompt，支持自定义实体类型
    """
    # 获取启用的实体类型配置
    entity_types = self.config_manager.get_entity_types()

    type_descriptions = []
    for type_key, type_config in entity_types.items():
        if type_config.get('enabled', True):
            name = type_config.get('name', type_key)
            description = type_config.get('description', '')
            type_descriptions.append(f"- {type_key}: {name}（{description}）")

    # 如果没有启用的实体类型，使用默认类型
    if not type_descriptions:
        type_descriptions = [
            "- PERSON: 人名（如：张三、李四）",
            "- ORG: 组织机构（如：阿里巴巴、清华大学）",
            "- LOC: 地点（如：北京、上海）",
```

#### 4.3.3 并行处理

```python
def extract_document_entities_parallel(self, document_id: int,
                                      max_workers: int = 4,
                                      batch_size: int = 10) -> Dict[str, Any]:
    """
    并行处理文档的所有片段

    Args:
        document_id: 文档ID
        max_workers: 并行工作线程数
        batch_size: 每批处理的片段数

    Returns:
        {"document_id": int, "total_chunks": int, "completed": int, "failed": int}
    """
    # 获取文档的所有片段
    chunks = self.db.query(DocumentChunk).filter(
        DocumentChunk.document_id == document_id
    ).all()

    total = len(chunks)
    completed = 0
    failed = 0

    # 使用线程池并行处理
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        # 提交所有任务
        future_to_chunk = {
            executor.submit(self.extract_chunk_entities, chunk.id): chunk
            for chunk in chunks
        }

        # 处理结果
        for future in as_completed(future_to_chunk):
            chunk = future_to_chunk[future]
            try:
                result = future.result()
                if "error" in result:
                    failed += 1
                else:
                    completed += 1
            except Exception as e:
                failed += 1

    return {
        "document_id": document_id,
        "total_chunks": total,
        "completed": completed,
        "failed": failed
    }
```

### 4.4 实体聚合流程

#### 4.4.1 文档级实体聚合

**核心类：** `DocumentEntityService`

**聚合算法：**
```python
def aggregate_chunk_entities(self, document_id: int) -> Dict[str, Any]:
    """
    将文档的所有片段级实体聚合成文档级实体

    算法：
    1. 按实体类型分组
    2. 计算相似度矩阵
    3. 层次聚类
    4. 为每个聚类创建文档级实体
    """
    # 1. 获取所有片段级实体
    chunk_entities = self.db.query(ChunkEntity).filter(
        ChunkEntity.document_id == document_id
    ).all()

    # 2. 按类型分组
    entity_groups = self._group_by_type(chunk_entities)

    # 3. 对每个类型组进行聚合
    for entity_type, type_entities in entity_groups.items():
        clusters = self._hierarchical_clustering(type_entities)
        for cluster in clusters:
            self._create_document_entity(document_id, entity_type, cluster)

def _hierarchical_clustering(self, entities: List[ChunkEntity]) -> List[List[ChunkEntity]]:
    """
    层次聚类算法

    步骤：
    1. 计算实体间相似度矩阵
    2. 初始化：每个实体为一个簇
    3. 迭代合并最相似的簇
    4. 直到相似度低于阈值
    """
    similarity_matrix = self._compute_similarity_matrix(entities)
    
    # 初始化簇
    clusters = [[i] for i in range(len(entities))]
    
    # 合并相似簇
    while len(clusters) > 1:
        best_sim = 0
        best_pair = (0, 1)
        
        # 找到最相似的一对簇
        for i in range(len(clusters)):
            for j in range(i + 1, len(clusters)):
                sim = self._cluster_similarity(clusters[i], clusters[j], similarity_matrix)
                if sim > best_sim:
                    best_sim = sim
                    best_pair = (i, j)
        
        # 如果最相似的簇相似度超过阈值，合并
        if best_sim >= self.text_similarity_threshold:
            i, j = best_pair
            clusters[i] = clusters[i] + clusters[j]
            clusters.pop(j)
        else:
            break
    
    return clusters
```

#### 4.4.2 知识库级实体对齐

**核心类：** `EntityAlignmentService`

**对齐流程：**
```python
def align_entities_for_kb(self, knowledge_base_id: int) -> Dict[str, Any]:
    """
    对指定知识库的所有文档实体进行对齐

    使用多维度相似度：
    - 文本相似度 (30%): Jaro-Winkler
    - BERT语义相似度 (40%): 基于BERT的语义理解
    - 传统语义相似度 (10%): 基于词重叠
    - 上下文相似度 (20%): 文档相关性
    """
    # 1. 获取知识库内所有文档实体
    entities = self.db.query(DocumentEntity).join(
        KnowledgeDocument
    ).filter(
        KnowledgeDocument.knowledge_base_id == knowledge_base_id
    ).all()

    # 2. 按实体类型分组
    entity_groups = self._group_by_type(entities)

    # 3. 对每个类型组进行对齐
    for entity_type, type_entities in entity_groups.items():
        kb_entities, aligned_count = self._align_entity_group(
            knowledge_base_id, entity_type, type_entities
        )

def _compute_similarity_matrix(self, entities: List[DocumentEntity]) -> np.ndarray:
    """
    计算实体间的相似度矩阵
    """
    n = len(entities)
    similarity = np.zeros((n, n))

    # 如果BERT对齐器可用，计算BERT语义相似度
    bert_similarity = None
    if self.bert_aligner:
        try:
            bert_similarity = self._compute_bert_similarity_matrix(entities)
        except Exception as e:
            logger.warning(f"BERT相似度计算失败: {e}")

    for i in range(n):
        for j in range(i + 1, n):
            # 文本相似度
            text_sim = self._text_similarity(
                entities[i].entity_text,
                entities[j].entity_text
            )

            # BERT语义相似度
            bert_sim = bert_similarity[i][j] if bert_similarity else 0.0

            # 传统语义相似度
            semantic_sim = self._semantic_similarity(
                entities[i].entity_text,
                entities[j].entity_text
            )

            # 上下文相似度
            context_sim = self._context_similarity(
                entities[i].document_id,
                entities[j].document_id
            )

            # 加权组合
            if bert_similarity is not None:
                similarity[i][j] = similarity[j][i] = (
                    0.30 * text_sim +
                    0.40 * bert_sim +
                    0.10 * semantic_sim +
                    0.20 * context_sim
                )
            else:
                similarity[i][j] = similarity[j][i] = (
                    0.40 * text_sim +
                    0.40 * semantic_sim +
                    0.20 * context_sim
                )

    return similarity
```

```python
def _hierarchical_clustering(self, entities, similarity_matrix):
    """
    当前算法复杂度：O(n³)
    
    问题：
    1. 双重循环找最相似簇对
    2. 每次合并后重新计算
    3. 文档实体数量大时性能急剧下降
    """
    clusters = [[i] for i in range(len(entities))]
    
    while len(clusters) > 1:
        best_sim = 0
        best_pair = (0, 1)
        
        # O(n²) 查找
        for i in range(len(clusters)):
            for j in range(i + 1, len(clusters)):
                sim = self._cluster_similarity(clusters[i], clusters[j], similarity_matrix)
                if sim > best_sim:
                    best_sim = sim
                    best_pair = (i, j)
```

```python
encodings = ['utf-8', 'gbk', 'gb18030', 'gb2312', 'latin-1']
```
- **问题**：优先尝试UTF-8，但中文文档多为GBK编码
- **影响**：中文文本文件解析时可能先失败再回退
- **建议**：根据文件头BOM或内容特征智能选择编码

**4. 大文件处理无流式读取**
```python
# 问题：一次性读取整个文件
doc = docx.Document(file_path)
text = ""
for paragraph in doc.paragraphs:
    text += paragraph.text + "\n"
```
- **问题**：大文档一次性加载到内存
- **影响**：可能导致内存溢出
- **建议**：实现流式处理或分块读取机制

### 8.3 文本清理流程问题

#### 8.3.1 代码位置
- [advanced_text_processor.py](file:///e:/PY/CODES/py%20copilot%20IV/backend/app/services/knowledge/core/advanced_text_processor.py#L38-L48)

#### 8.3.2 发现的问题

**1. 清理规则过于简单**
```python
def clean_text(self, text: str) -> str:
    if text is None:
        return ""
    # 移除多余的空格和换行
    text = re.sub(r'\s+', ' ', text)
    # 移除特殊字符，保留中文、英文、数字和基本标点
    text = re.sub(r'[^\w\s\u4e00-\u9fff.,!?;:()\[\]{}]', '', text)
    return text.strip()
```
- **问题**：
  - 正则表达式 `[^\w\s\u4e00-\u9fff...]` 会误删有用符号
  - 未处理HTML标签、Markdown标记等
  - 未处理页眉页脚、页码等文档噪声
- **影响**：清理后的文本可能丢失重要格式信息
- **建议**：
  - 增加HTML/Markdown标签清理
  - 增加页眉页脚检测与移除
  - 保留有意义的格式标记

**2. 缺乏配置化能力**
- **问题**：清理规则硬编码，无法根据文档类型调整
- **影响**：不同类型文档需要不同的清理策略
- **建议**：提供清理规则配置接口，支持按文档类型定制

### 8.4 文档分块流程问题

#### 8.4.1 代码位置
- [advanced_text_processor.py](file:///e:/PY/CODES/py%20copilot%20IV/backend/app/services/knowledge/core/advanced_text_processor.py#L230-L250)
- [llm_text_processor.py](file:///e:/PY/CODES/py%20copilot%20IV/backend/app/services/knowledge/llm_text_processor.py)

#### 8.4.2 发现的问题

**1. 语义分块依赖LLM，成本高**
```python
async def semantic_chunking(self, text: str, max_chunk_size: int = 1000,
                     min_chunk_size: int = 200, overlap: int = 30,
                     semantic_threshold: float = 0.7) -> List[str]:
    return await self.llm_text_processor.semantic_chunking(text, max_chunk_size, min_chunk_size, overlap)
```
- **问题**：所有分块操作都调用LLM
- **影响**：处理成本极高，响应时间长
- **建议**：
  - 实现基于规则的分块作为默认方案
  - LLM分块作为可选的高级功能
  - 增加分块缓存机制

**2. 分块重叠策略固定**
```python
overlap: int = 30  # 重叠窗口大小，默认30以保证上下文连贯同时减少重复
```
- **问题**：重叠大小固定为30字符，未考虑语义边界
- **影响**：可能在句子中间截断，影响语义连贯性
- **建议**：按句子或段落边界计算重叠，而非固定字符数

**3. 分块大小控制不精确**
- **问题**：`max_chunk_size` 按字符数计算，但向量化时按token计算
- **影响**：可能超出模型token限制
- **建议**：使用tiktoken等库精确计算token数
