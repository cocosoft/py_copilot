## 二、适配器架构设计

### 2.1 适配器继承关系

```
BaseCapability (抽象基类)
    ├── SkillCapability
    ├── ToolCapability
    └── MCPCapability
```

### 2.2 注册表架构

```
SkillRegistry ── 管理 SkillCapability 实例
ToolRegistry  ── 管理 ToolCapability 实例
MCPRegistry   ── 管理 MCPCapability 实例
```

### 2.3 数据流

```
用户输入
    ↓
Capability执行请求
    ↓
适配器转换
    ├── SkillCapability → SkillExecutionEngine
    ├── ToolCapability  → ToolIntegrationService / MCPClient / Handler
    └── MCPCapability   → MCPClient
    ↓
原始系统执行
    ↓
结果转换 → CapabilityResult
    ↓
返回给用户
```

## 四、关键技术点

### 4.1 Skill级别自动判断

```python
def _determine_level(self, skill: Skill) -> CapabilityLevel:
    if skill.execution_flow and len(skill.execution_flow) > 0:
        return CapabilityLevel.WORKFLOW
    if skill.dependencies and len(skill.dependencies) > 0:
        return CapabilityLevel.COMPOSITE
    return CapabilityLevel.ATOMIC
```

### 4.2 Tool多类型执行

```python
async def _do_execute(self, input_data, context):
    if self.tool.tool_type == "mcp":
        return await self._execute_mcp(input_data, context)
    elif self.tool.tool_type == "official":
        return await self._execute_official(input_data, context)
    else:  # local
        return await self._execute_local(input_data, context)
```

### 4.3 本地处理函数注册

```python
@classmethod
def register_local_handler(cls, tool_name: str, handler: Callable):
    cls._local_handlers[tool_name] = handler
```
